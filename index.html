Resorto GitHub

Resorto is a resort management game I created in Unity with C#. The sprites were created using Gimp and Aseprite. In this game, you play as a resort manager in charge of attracting tourists to a deserted island.


Procedual island generation

Script:

The island terrain is generated randomly using Perlin noise, with the darker areas of the noise representing hills and mountains and white areas representing the ocean.


Terrain modification &
structures placement.

Script:

The game uses a 2D array to store tile information like terrain, terrain height, zone, and structures on the tile. The player has the ability to modify the terrain and place structures.


Inventory

Script:

The player has an inventory to store items. Items can be moved around the inventory smoothly, and can also be moved to storages.

Pathfinding

Script:

The tourists move around the island using A* pathfinding. When a tourist decides where they want to go, the shortest path to the destination will be calculated while avoiding terrain, water, and obstacles.


Zoning

The player can create zones around the island to dedicate spaces for certain activities. For example, if a fishing zone is created, the tourists will only go to that zone to fish.


Fishing

Script:

Fish randomly spawns around the ocean. The player and tourists can take part in fishing.

The fishing line animation is created using Bezier curves. The 3 points of the curve changes during different parts of the animation to create a natural-looking casting animation.

The fishing line can be charged to cast farther and can be cast in different directions. And the bezier line renderer will change the rendering accordingly.

Tourist schedules and traits

Script:

Each tourist will have different schedules. Like what time they go to bed, what day they leave the island, or when they eat.

A clock event system is set up so that it can be subscribed and unsubscribed to. Tourists could subscribe to any given time, such as 7:32pm. And when that time is reached, all subscribers (tourists in this case) of that time will be notified that the time has reached, and they would start doing their scheduled activities.

Each tourist arriving on the island has random traits. Some will prefer different activities such as fishing, and will take part in that activity more frequently. 


Character customization

Script:

The player can customize their looks using the character customization menu, and cosmetic items like hats.

Cellular automata GitHub

In Conway's game of life, a cell can be either dead or alive, and its next state is determined by how many cells are alive in its 8 neighbouring cells. In this cellular automata simulator I created in C#, a cell can take in any float between 0 and 1, and the neighbouring cells to check can be changed.

Instead of making the cells alive or dead, rules can be created to either add or subtract some value from the cells depending on how many neighbouring cells are alive.

I also created a React app to configure rules. The neighbours each cell checks and the color can be configured in the UI. The color of cells can be configured to be a gradient, such that the cell's value (between 0 and 1) corresponds to a certain gradient value, and the simulator will show that color for that cell. The rule can be exported to json and loaded into the simulator.
